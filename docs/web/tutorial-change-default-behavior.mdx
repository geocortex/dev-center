Many of the behaviors of Geocortex Web's built-in [components](sdk-components-overview.mdx) are configurable through the [app config](configuration-app-config-getting-started.mdx).

:::tip
Many behaviors which are configurable through the app config are more easily configurable by using the [Geocortex Web Designer](https://apps.geocortex.com/webviewer/designer/). App config should be used if you need to modify a property not available in the app config, or use a custom command as part of a [command chain](configuration-commands-operations.mdx#configuring-commands-and-operations).
:::

## Change the Behavior of the Bookmarks Component

The following example demonstrates changing the `onClick` property of the bookmarks model. The default value of this property is the singular command, `map.zoom-to-viewpoint`. This property is not available to edit through the Geocortex Web Designer, but can be edited through the use of app config.

First, let's create a basic map with a bookmarks component using the layout and app config. Start up the application and add a few bookmarks.

<Tabs
  defaultValue="layout"
  values={[
    { label: 'Layout', value: 'layout', },
    { label: 'App Config', value: 'app', },
  ]
}>
<TabItem value="app">

```json title="app/app.json"
{
    "schemaVersion": "1.0",
    "items": []
}
```

</TabItem>
<TabItem value="layout">

```xml title="app/layout.xml"
<?xml version="1.0" encoding="UTF-8"?>
<layout xmlns="https://geocortex.com/layout/v1" xmlns:custom="your.custom.namespace" xmlns:web="https://geocortex.com/layout/web/v1">
    <map>
        <web:bookmarks/>
    </map>
</layout>
```

</TabItem>
</Tabs>

Next, add a [app item](configuration-app-config-getting-started.mdx#anatomy-of-an-app-config) to the configuration for the bookmarks component.

<Tabs
  defaultValue="layout"
  values={[
    { label: 'Layout', value: 'layout', },
    { label: 'App Config', value: 'app', },
  ]
}>
<TabItem value="app">

```json title="app/app.json"
{
    "schemaVersion": "1.0",
    "items": [
        {
            "id": "bookmarks-config",
            "$type": "bookmarks"
        }
    ]
}
```

</TabItem>
<TabItem value="layout">

```xml title="app/layout.xml"
<?xml version="1.0" encoding="UTF-8"?>
<layout xmlns="https://geocortex.com/layout/v1" xmlns:custom="your.custom.namespace" xmlns:web="https://geocortex.com/layout/web/v1">
    <map>
        <web:bookmarks config="bookmarks-config"/>
    </map>
</layout>
```

</TabItem>
</Tabs>

Finally, modify the `onClick` property of the bookmarks app item to run the commands `map.zoom-to-viewpoint` and `ui.alert`.

<Tabs
  defaultValue="config"
  values={[
    { label: 'Layout', value: 'layout', },
    { label: 'App Config', value: 'config', },
    { label: 'UI', value: 'ui', },
  ]
}>
<TabItem value="layout">

```xml title="app/layout.xml"
<?xml version="1.0" encoding="UTF-8"?>
<layout xmlns="https://geocortex.com/layout/v1" xmlns:custom="your.custom.namespace" xmlns:web="https://geocortex.com/layout/web/v1">
    <map>
        <web:bookmarks config="bookmarks-config"/>
    </map>
</layout>
```

</TabItem>
<TabItem value="config">

```json title="app/app.json"
{
    "schemaVersion": "1.0",
    "items": [
        {
            "id": "bookmarks-config",
            "$type": "bookmarks",
            "onClick": [
                "map.zoom-to-viewpoint",
                {
                    "name": "ui.alert",
                    "arguments": {
                        "message": "Message from config."
                    }
                }
            ]
        }
    ]
}
```

</TabItem>
<TabItem value="ui">
<img src={useBaseUrl("img/web-customize-bookmarks-component.png")}/>
</TabItem>
</Tabs>

Congratulations! You just changed the default behavior for a built in component using config. All components can be configured through the app config JSON in this manner.
