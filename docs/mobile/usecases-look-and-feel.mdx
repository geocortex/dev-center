Geocortex Mobile's appearance is defined by two concepts: theme and [layout](concepts-layout.mdx).

The theme controls the color scheme of the app. The color scheme for different themes is defined in branding component of the the [app config](concepts-app-config.mdx). This example is built around [editing the app JSON of the Quickstart](sdk-edit-layout-app-config.mdx) project.

## Theming with the Branding Component

This example shows two pre-defined themes that are included with Mobile, light and dark. Custom themes can be added to Mobile's included themes, and selected by changing `activeTheme` property of the branding component.
The branding component can be found and edited in the [App Config](sdk-edit-layout-app-config.mdx).

Color properties can be excluded from a theme; The application will choose a default color instead.

## Using Theme Colors in Custom Components

When creating custom components, you may need to style your component. If you use the [Geocortex Mobile UI Elements](api-mobile-ui-elements.mdx) to style your XAML elements, your element will be themed automatically. If you need to use plain XAML elements, the various theming colors can be used in XAML styling.

```xml
<Style x:Key="InnerDetailsFrameStyle" TargetType="Frame">
  ...
  <Setter Property="BackgroundColor" Value="{DynamicResource SecondaryBackgroundColor}"></Setter>
  ...
</Style>
```

## Changing the Visual Layout of an Application

An app's layout defines how elements are visually organized in a Geocortex Mobile Application. [This article](concepts-layout.mdx) will teach you how to configure an app's layout and how to leverage it to customize the appearance of Geocortex Mobile. Further styling within the layout can be done using [presentation attributes](api-layout.mdx#presentation-attributes).
