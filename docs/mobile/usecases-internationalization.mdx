Geocortex Mobile, being a Xamarin Forms Application, comes with built in [support for full internationalization](https://docs.microsoft.com/en-us/xamarin/xamarin-forms/app-fundamentals/localization/text?pivots=windows) of existing and new language strings.

## Specifying a Default Language

Xamarin forms describes how to [specify a default language for your app](https://docs.microsoft.com/en-us/xamarin/xamarin-forms/app-fundamentals/localization/text?pivots=windows#specify-the-default-culture).

## Adding New Cultures for Existing Language Strings

The [Quickstart](sdk-getting-started.mdx) project comes with multiple `AppResources` files in the [root of the platform agnostic project](https://github.com/geocortex/vertigis-mobile-quickstart/tree/master/App1/App1) to provide an example of adding a culture to your application.

## Providing Language Strings for a Component

The [SDK Samples](sdk-samples.mdx) project includes a [internationalization example](https://github.com/geocortex/vertigis-mobile-samples/tree/master/Geocortex.Mobile.Samples/Geocortex.Mobile.Samples/Samples/Conceptual/Internationalization) which demonstrates how to configure resource files on a component level.

:::note
By convention, all language string keys take the format `language-kebab-cased-description`.
:::

## Language Strings and App Config

All language strings used for properties in the [app config](concepts-app-config.mdx), like "title" or "description", are automatically translated by the application.

```json
{
    "items": [
        ...
        {
            "id": "zoom-to-extent",
            "iconId": "zoom-extent",
            "title": "language-zoom-extent-title",
            "description": "language-zoom-extent-description",
            "action": "map.zoom-to-layer-extent",
            "$type": "menu-item"
        },
        ...
    ]
}
```
